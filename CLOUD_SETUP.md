# 云开发部署说明

## 当前状态

项目已经完成以下准备：

- 已接入云开发初始化
- 已提供 `auth` 和 `relationship` 两个云函数
- 已支持图片上传到云存储
- 已在首次鉴权时自动初始化以下集合：
  - `users`
  - `space`
  - `moments`
  - `anniversaries`
  - `wishes`

## 你现在需要做的事

### 1. 打开微信开发者工具

导入项目目录：

`C:/Users/tianzhaoyun/Desktop/DI`

### 2. 开通云开发环境

在微信开发者工具顶部找到“云开发”。

如果还没有环境：
- 创建一个新的云开发环境
- 记下环境 ID

当前项目配置使用的是动态环境：
- `config/index.js` 中 `cloudEnvId` 为 `DYNAMIC_CURRENT_ENV`

这意味着项目会直接使用当前开发者工具里选中的云环境。

### 3. 部署云函数

在微信开发者工具中进入：

- `cloudfunctions/auth`
- `cloudfunctions/relationship`

对这两个目录分别执行：
- 右键
- 选择“上传并部署：云端安装依赖”

## 数据库集合说明

需要先在云开发控制台中手动创建以下集合：

- `users`
- `space`
- `moments`
- `anniversaries`
- `wishes`

创建后不需要预填数据。

首次成功调用 `auth` 云函数后，项目会自动写入 `space/main` 文档，并初始化后续使用的数据结构。

## 首次初始化流程

部署完云函数后：

1. 打开小程序
2. 首次进入会先进入鉴权页 `pages/auth/index`
3. `auth` 云函数会自动执行白名单绑定与集合初始化
4. 首位进入的用户会成为 `owner`
5. 第二位进入的用户会成为 `partner`
6. 超过两位后会被拒绝进入

## 白名单规则

当前规则为：

- 前两位首次进入空间的微信账号会自动加入白名单
- 第三位及以后会被拒绝
- 白名单数据存放于 `users` 集合

## 如果你要手动调整白名单

在云数据库 `users` 集合中可以修改：

- `nickname`
- `avatarUrl`
- `role`
- `openid`

建议不要随意删除首位用户，否则会影响空间绑定关系。
