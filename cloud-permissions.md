# 云数据库权限规则建议

当前项目的核心数据写入都已经收口到 `auth` 和 `relationship` 云函数里，客户端默认不应该直连数据库做写操作。

建议在微信云开发控制台里按下面原则配置数据库权限：

## 总原则

- `users`：仅云函数可写；前端禁止直接改白名单成员。
- `space`：仅云函数可读写；前端不要直接改空间主文档。
- `moments`：仅云函数可写；读可以按业务需要开放给已登录成员。
- `anniversaries`：仅云函数可写。
- `wishes`：仅云函数可写。

## 推荐做法

1. 数据库权限改成“仅创建者可读写”并不适合这个项目。
2. 更适合的方式是：数据库侧尽量收紧，业务写操作全部走云函数。
3. 云函数内部统一通过 `ensureAuthorizedUser()` 做两人白名单校验。
4. 删除回忆、删除评论这类敏感操作继续在云函数里做 `createUserId / userId` 判断。

## 当前代码里已经补齐的权限点

- 所有业务动作都要求用户已经存在于 `users` 集合。
- 删除回忆时会校验创建人。
- 删除评论时会校验评论创建人或回忆创建人。
- 点赞、评论、宠物互动、提醒设置、空间修改都只允许已授权成员调用。

## 仍需手动配置的控制台项

- 数据库集合权限规则。
- 云函数的上线部署。
- 如果后续要把“纪念日提醒”升级成订阅消息：还需要模板 ID、用户订阅授权和定时触发器。
