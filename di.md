# 我们的空间开发文档

## 1. 项目定位

- 项目名称：我们的空间
- 项目形态：微信小程序 + 云开发
- 使用对象：仅限两位成员的私密双人空间
- 当前目标：围绕回忆记录（最主要）、纪念日（主要）、心愿（主要）、空间管理和宠物互动（次要）持续做二开

## 2. 当前页面与入口

### 2.1 底部 Tab

- 首页
- 时光墙
- 纪念日
- 宠物岛
- 心愿单

### 2.2 非 Tab 页面

- 登录授权页：`pages/auth/index`
- 空间设置页：`pages/my/index`
- 发布回忆页：`pages/release/index`
- 回忆详情页：`pages/moment/detail`
- 日历日记页：`pages/diary/index`

说明：旧的“纪念与心愿合并页”已从路由中移除，纪念日和心愿能力以独立页面为准。

## 3. 数据与云开发

### 3.1 已接入集合

| 集合名 | 用途 |
| --- | --- |
| `users` | 双人白名单、成员身份信息 |
| `space` | 空间基础信息、在一起日期、提醒设置、宠物状态 |
| `moments` | 时光墙内容、图片、点赞、评论、地点信息 |
| `anniversaries` | 自定义纪念日 |
| `wishes` | 心愿清单 |

### 3.2 已接入云能力

- `auth` 云函数：登录白名单校验、空间初始化
- `relationship` 云函数：空间内容的增删改查
- 图片上传：已用于时光墙发布

## 4.4 UI 细节规范

- 表单输入框和主操作按钮之间必须保留明确的纵向呼吸感，避免紧贴导致界面发闷。
- 涉及多个表单项之间的间距、分组上下关系、按钮与输入区之间的布局时，优先由页面级容器统一控制，例如使用外层 `flex + gap`、独立包裹层或明确的块级间距；不要把页面排版依赖在组件内部类名上，否则容易出现“代码已修改但界面无变化”的假象。
- 使用自定义组件（如 `warm-input`、`warm-button`）时，若视觉调整未生效，必须先确认样式究竟作用在页面宿主、组件根节点还是组件内部节点；布局问题先查组件边界，再决定改页面容器还是改组件实现，不允许直接假设页面样式一定能穿透组件内部结构。
- 进度条、说明文本和删除/次操作按钮之间要避免过宽留白，优先通过更紧凑的排版关系提升整体感。
- 像心愿花园这类情绪化页面，布局尽量做成多行多列、有层次的场景，不要只停留在普通信息卡片堆叠。
- 底部双操作按钮优先使用对称布局，主次关系通过色彩和通透度区分，不要依赖不等宽造成失衡。
- 悬浮在页面内容之上、卡片之上或场景中的按钮，统一使用“半透明糖玻璃胶囊按钮”这套样式，不再退回普通纯色矩形按钮。
- 这套悬浮按钮样式的最低要求是：圆润胶囊外轮廓、半透明或玻璃感材质、柔和顶部高光、细腻渐变过渡、主次按钮通过透明度和发光强弱区分，而不是只通过大小失衡或生硬色块区分。
- 悬浮主按钮应保持柔和发光感和更明确的视觉聚焦，但不能出现高光过硬、颜色过满、像实体塑料块一样压在页面上的问题；悬浮次按钮应更轻、更透，接近半透明玻璃质感。
- 固定在页面底部的操作区必须提前为滚动内容预留足够底部空间，避免最后一组表单项、选项或说明文字被遮挡。
- 固定按钮不得直接复用悬浮糖玻璃样式，必须使用另一套更稳定、更贴地的固定操作样式，避免页面底部像飘着两块糖玻璃。
- 固定按钮的推荐形态是“托盘 + 按钮本体”两层结构：外层托盘负责承接页面底部并与内容收口，内层按钮负责主次操作表达；托盘允许轻微半透明，但整体必须比悬浮按钮更实、更稳。
- 固定按钮的主按钮应采用更高密度的暖粉或暖红渐变，透明度明显低于悬浮按钮，文字对比更强，高光更窄更收敛，避免继续做成整块通透发亮的糖玻璃。
- 固定按钮的次按钮应采用奶白、雾粉或浅米色实体感底面，依靠清晰边界、细描边和弱阴影建立层次，不再靠大面积透底来区分主次。
- 固定按钮的圆角仍应柔和，但整体厚度感、边界感和按压反馈必须更明确；可保留轻微顶部柔光与细腻渐变，不可保留大面积悬浮泛光、强透底或过度背景渗透。
- 若按钮固定在页面底部，外层承托区域应使用偏低饱和、偏浅暖色的结构托盘，并通过圆角、上缘柔光、下缘投影完成“固定、承接、收口”的作用；不要继续套用悬浮按钮那种弱托盘、强漂浮的处理方式。
- 固定操作区内若存在双按钮，优先采用等宽对称布局；主按钮负责聚焦，次按钮负责平衡，但两者都应落在同一托盘体系内，不要像两个独立漂浮物。
- 后续新页面或改版页面涉及操作按钮时，必须先判断按钮属于“悬浮按钮”还是“固定按钮”：悬浮按钮沿用当前这套半透明糖玻璃语言，固定按钮单独设计，不得混用。

## 5. 当前功能拆分

### 5.1 首页

- 展示在一起天数、回忆数和最近心情
- 展示最近回忆和地点信息
- 支持直接进入发布回忆

### 5.2 时光墙

- 瀑布式回忆内容浏览
- 回忆发布、编辑展示优化、多图预览
- 点赞、评论、地点和详情查看
- 提供进入日历日记的入口

### 5.3 纪念日

- 在一起日期维护
- 自定义纪念日管理
- 100 天里程碑与年度进度展示
- 应用内提醒提示和提前天数切换

### 5.4 心愿单

- 心愿录入
- 完成状态切换
- 已完成 / 待完成展示
- 页面统计信息展示

### 5.5 宠物岛

- 双宠物状态展示
- 投喂、贴贴、等级成长
- 基础皮肤切换

### 5.6 日历日记

- 月视图查看记录密度
- 查看某一天的已发布内容
- 从指定日期发起记录

### 5.7 空间设置

- 空间资料维护
- 成员昵称维护
- 一键清空空间数据
