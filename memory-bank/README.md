# Memory Bank

这个目录用于在项目根目录长期保存项目上下文，避免二开时重复梳理。

## 目录说明

- `project-overview.md`：项目定位、核心目标、当前页面入口
- `tech-stack.md`：技术栈、依赖、运行方式、项目约束
- `architecture.md`：前后端结构、数据流、模块边界
- `decisions.md`：关键架构决策与明确约束
- `operations.md`：云开发部署、权限、上线前检查项

## 维护规则

1. 每次新增核心模块、云函数能力或页面入口时，同步更新对应文件。
2. 需求结论优先沉淀到 `decisions.md`，避免只留在聊天记录里。
3. 与部署、数据库权限、云函数上线相关的信息统一写到 `operations.md`。
4. 如果页面结构或项目定位发生变化，优先更新 `project-overview.md`。
