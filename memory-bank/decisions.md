# Decisions

## 产品决策

### 1. 主题以“记录”为主

- 回忆记录是项目第一优先级
- 纪念日与心愿是主线辅助模块
- 宠物岛是次要模块，不应喧宾夺主

### 2. 项目保持双人空间

- 当前明确只服务两位成员
- 不做多人协作、多人邀请、群组空间扩展

### 3. 纪念日与心愿单拆分维护

- 旧的合并页已经移除
- 后续以独立页面持续迭代，避免页面职责混杂

## 技术决策

### 4. 所有核心写操作走云函数

- 前端不直接写核心集合
- 权限校验集中放在云函数里处理
- 删除类敏感操作继续保留创建人校验

### 5. 启动时先云初始化，再做鉴权

- `app.js` 负责启动期的云环境初始化与登录校验
- 页面通过统一授权工具兜底，避免各页各自实现一套逻辑

### 6. 视觉语言统一为暖色梦幻风格

- 背景氛围偏柔和、空灵、稍带甜感
- 点缀尽量放两侧和底部
- 不能让装饰压住中间卡片和主信息

## 运维决策

### 7. 改前端后要同步考虑云函数部署

- 当前项目里，前端改完但云函数没重部署是高概率问题
- 涉及 `space.startDate`、点赞、评论、地点、提醒、宠物、日历相关逻辑时尤其要注意

### 8. 数据库权限默认收紧

- 数据库权限以控制台配置和云函数授权为主
- 不依赖前端自行约束权限
