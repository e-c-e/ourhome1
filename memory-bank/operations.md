# Operations

## 云开发上线注意事项

### 1. 云函数部署

- 核心云函数：`auth`、`relationship`
- 每次修改涉及数据写入、鉴权或结构变更的逻辑后，都应重新部署相关云函数

### 2. 数据库权限

当前建议：

- `users`：仅云函数可写
- `space`：仅云函数可读写
- `moments`：仅云函数可写，读按业务开放给已授权成员
- `anniversaries`：仅云函数可写
- `wishes`：仅云函数可写

原则：数据库侧尽量收紧，业务写操作统一走云函数。

### 3. 当前已知高频问题

- 前端改好了，但云函数没有重新部署，导致能力不生效
- 在一起日期写入后，如果回读异常，优先怀疑 `relationship` 云函数未更新
- 新增能力后如果控制台权限没同步收紧，会留下安全隐患

## 当前上线前检查清单

1. 确认 `auth`、`relationship` 已重新部署
2. 确认数据库权限已经按 `cloud-permissions.md` 配置
3. 确认白名单用户可正常登录，非白名单被拦截
4. 确认 `space.startDate` 可正确保存并回读
5. 确认时光墙发布、点赞、评论、地点、多图预览正常
6. 确认纪念日进度条、提醒设置、心愿单操作正常
